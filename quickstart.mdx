title: "Quickstart"
description: "Add Repo Security Bot to your repository in under 5 minutes"
---

Repo Security Bot is a plug-and-play GitHub Security Scanner that automatically runs five industry-standard security tools — **Gitleaks**, **Bandit**, **Checkov**, **Nuclei**, and **Dockle** — on every push, pull request, or scheduled scan. It aggregates all results into a clean `security-summary.md` and automatically opens a GitHub Issue with the full findings.

<Steps>

<Step title="Create the workflow file">

In your repository, create the file `.github/workflows/run-security-bot.yml` with the following content:

```yaml .github/workflows/run-security-bot.yml
name: "Run Security Bot"

on:
  push:
    branches: [ main ]   # change branch if needed
  pull_request:
  schedule:
    - cron: "0 12 * * *"  # OPTIONAL: run daily at 12:00 UTC
  workflow_dispatch:      # manual trigger

permissions:
  contents: read
  issues: write
  checks: write
  actions: read
  pull-requests: write

jobs:
  call-bot:
    uses: arinagrawal05/repo-security-bot/.github/workflows/repo-security-bot.yml@main
    
    secrets: inherit    # passes your repo secrets (SNYK_TOKEN, etc.)
```

</Step>

<Step title="Understand the permissions">

The workflow requests the following permissions. Each is required for a specific part of the bot's operation:

| Permission | Why it's needed |
| --- | --- |
| `contents: read` | Allows the bot to clone your repository code so scanners can analyze it |
| `issues: write` | Allows the bot to create and edit GitHub Issues containing the security findings report |
| `checks: write` | Allows the bot to upload check results so scan status is visible on commits and pull requests |
| `actions: read` | Allows the bot to read workflow artifacts produced by individual scanner jobs |
| `pull-requests: write` | Allows the bot to post comments directly on pull requests |

</Step>

<Step title="Understand the trigger types">

The workflow runs on four different trigger types:

- **`push`** — Runs on every push to `main` (or whichever branch you configure). Catches issues as code lands.
- **`pull_request`** — Runs automatically when a pull request is opened or updated. Surfaces findings before code is merged.
- **`schedule`** — Runs on a cron schedule (`0 12 * * *` by default, which is daily at 12:00 UTC). Catches drift or newly published vulnerabilities against your existing codebase.
- **`workflow_dispatch`** — Allows you to trigger the bot manually from the GitHub Actions tab at any time.

</Step>

<Step title="Commit and push">

Commit the workflow file and push it to your default branch:

```bash
git add .github/workflows/run-security-bot.yml
git commit -m "Add Repo Security Bot workflow"
git push
```

The bot will start running on the push event immediately.

<Tip>
After pushing, open the **Actions** tab in your repository to watch the workflow run in real time. Each scanner job runs in parallel, and the summary job waits for all of them to complete before generating the report.
</Tip>

</Step>

</Steps>

<Note>
The `secrets: inherit` line passes all secrets defined in your repository to the reusable workflow. This is how the bot accesses tokens such as `SNYK_TOKEN` without you needing to redeclare them. If you do not have any additional secrets configured, this line is still safe to include — it simply passes nothing extra.
</Note>

## What happens next

Once the workflow runs, the bot executes five scanners in parallel:

1. **Gitleaks** scans for hardcoded secrets, API keys, credentials, and tokens
2. **Bandit** checks Python code for security issues such as injections, unsafe subprocess usage, and weak cryptography
3. **Checkov** audits infrastructure-as-code files (Terraform, CloudFormation, Kubernetes, Serverless) for misconfigurations
4. **Nuclei** runs community vulnerability templates covering CVEs, misconfigurations, and exposures
5. **Dockle** checks Docker images against best practices and CIS benchmarks

After all five scanners finish, a summary job collects their outputs and:

- Generates a `security-summary.md` combining all findings
- Uploads the following artifacts to the workflow run:
  ```
  security-summary.md
  gitleaks.json / gitleaks.md
  bandit.json / bandit.md
  checkov.json / checkov.md
  nuclei.json / nuclei.md
  dockle.json / dockle.md
  ```
- Opens a GitHub Issue titled **"Security Summary Report: YYYY-MM-DD"** containing the full report with scanner sections, markdown tables, severity counts, actionable findings, and recommended fixes

The issue is ready for triage immediately after the run completes.
