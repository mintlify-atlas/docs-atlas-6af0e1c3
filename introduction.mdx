## What is Repo Security Bot?

**Repo Security Bot** is a plug-and-play GitHub Actions security scanner that automatically runs **five industry-standard security tools** on every push, pull request, or scheduled scan.

It aggregates all findings into a clean `security-summary.md` and automatically creates a **GitHub Issue** containing the complete results — giving your team immediate, actionable vulnerability insights with zero manual effort.

## How it works

<Steps>
  <Step title="Add a workflow file to your repo">
    Create `.github/workflows/run-security-bot.yml` in your repository and call the reusable workflow hosted in `arinagrawal05/repo-security-bot`.
  </Step>
  <Step title="Bot runs on every push or PR">
    On every push to `main`, pull request, or scheduled cron trigger, the bot spins up five parallel scanner jobs.
  </Step>
  <Step title="Findings are aggregated">
    All five scanner results are collected, parsed, and merged into a single `security-summary.md` artifact.
  </Step>
  <Step title="GitHub Issue is created automatically">
    A GitHub Issue is opened in your repository with the full security report — severity counts, findings tables, and recommended fixes.
  </Step>
</Steps>

## Five integrated scanners

| Tool | What it detects |
|------|----------------|
| **Gitleaks** | Hardcoded secrets, API keys, credentials, tokens |
| **Bandit** | Python security issues: injections, unsafe subprocess, weak crypto |
| **Checkov** | IaC misconfigurations (Terraform, CloudFormation, Kubernetes) |
| **Nuclei** | Vulnerability scans using community templates (CVE, misconfig, exposures) |
| **Dockle** | Docker image best practices, hardening issues, CIS violations |

## Key features

  <Card title="Zero setup" icon="bolt">
    A single workflow file is all you need. No agents to install, no dashboards to configure.
  </Card>
  <Card title="Parallel scanning" icon="arrows-split-up-and-left">
    All five scanners run in parallel as independent GitHub Actions jobs.
  </Card>
  <Card title="Auto Issue creation" icon="circle-exclamation">
    Each scan run automatically opens a GitHub Issue with the aggregated findings.
  </Card>
  <Card title="Flexible triggers" icon="clock">
    Run on push, pull request, manual dispatch, or on a daily/weekly cron schedule.
  </Card>
  <Card title="Artifact exports" icon="box-archive">
    Clean Markdown reports are uploaded as workflow artifacts for every scan. Checkov also includes raw JSON results.
  </Card>
  <Card title="Secrets-safe reporting" icon="eye-slash">
    Gitleaks reports omit actual secret values — only rule ID, file, and line are reported.
  </Card>
</CardGroup>
